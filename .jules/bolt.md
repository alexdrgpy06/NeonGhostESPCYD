## 2024-06-01 - Optimizing Sprite Rendering with Run-Length fillRect
**Learning:** The `CreatureRenderer` draws 24x24 pixel art sprites by iterating over every pixel and issuing individual `fillRect` calls to the TFT display for each set pixel. Due to the high number of contiguous set pixels in most sprites, this results in significant driver overhead (many separate SPI transactions or API calls). Consolidating adjacent horizontal pixels into a single, wider `fillRect` acts like Run-Length Encoding and dramatically reduces call counts (measured 44 down to 14 in a simple benchmark for a 3-row block).
**Action:** When rendering pixel art or solid blocks on hardware displays, always scanline render by grouping contiguous identical pixels into a single draw command rather than drawing pixel-by-pixel.
