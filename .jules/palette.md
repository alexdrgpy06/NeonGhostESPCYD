## 2024-05-02 - Button Visual Feedback and strict touch bounds
**Learning:** Hardware touch interactions on ESP32 without haptics require strong, immediate visual cues (like momentarily redrawing with inverted colors using a short delay, ~50ms) to confirm the input was received. This prevents users from repeatedly tapping buttons, which improves interaction confidence and the overall UX flow. Furthermore, touch interaction bounds checking for buttons must explicitly verify both the upper and lower boundaries to accurately restrict active tap zones.
**Action:** When implementing or refactoring custom UI elements for physical TFT screens, ensure to include visual feedback (like color inversion or shape changes) tied directly to the touch event loop before executing the attached action logic. Also properly restrict the touch event bounding box.
