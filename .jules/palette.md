## 2025-01-20 - Physical Touchscreen Immediate Visual Feedback and Bounds Checking
**Learning:** Adding immediate visual feedback (e.g., momentary color inversion via a short `delay`) is crucial on physical touchscreens (like the CYD) because it assures users their tap was registered, reducing the tendency to repeatedly double-tap buttons. Additionally, omitting explicit lower bounds checking in a tap target handler can unintentionally convert the entire bottom portion of the screen into a clickable area for a small button, leading to accidental actions.
**Action:** Always implement a 'press-wait-release' drawing pattern using helper functions (like `drawSingleButton`) on physical displays and explicitly define both upper and lower boundaries (`ty >= BUTTON_Y && ty <= BUTTON_Y + BUTTON_H`) for interactive components.
